# FUTURE_CS_03 – API Security Risk Analysis

## 📌 Project Overview

This project was completed as part of the **Future Interns Cyber Security Internship – Task 3**. The objective is to perform a security assessment of a public REST API by analyzing its endpoints, HTTP responses, security headers, and overall security posture using industry-standard tools.

The assessment follows a structured methodology based on **OWASP API Security Top 10** best practices and documents the identified risks along with recommendations for improving API security.

> **Note:** This assessment was performed on the public **JSONPlaceholder API**, which is intended for testing and learning purposes only.

---

## 🎯 Objectives

- Analyze publicly accessible API endpoints.
- Perform basic host reconnaissance.
- Inspect HTTP request and response headers.
- Evaluate authentication and authorization mechanisms.
- Identify potential data exposure.
- Review API security against OWASP API Security Top 10.
- Document findings and recommend security improvements.

- HTTP response headers were successfully analyzed.
- Public data exposure is expected for the selected testing API.
- No sensitive information was exposed.
- Basic security headers were observed during analysis.
